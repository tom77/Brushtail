Brushtail Content Management Guide

Leave applications

Description

This module allows staff to apply for leave via the intranet. The manager is alerted by email of any leave applications. Staff can edit and update leave applications until they have been approved. Once approved, leave applications can only be edited by managers. The manager can see a summary of pending applications. When applications are approved, an email is sent back to the applicant, informing them of application status. Staff groups can be created to allow granular control over manager permissions.

Setup

  1. Create leave module via Intranet administration > Main menu
  2. Grant edit permissions to content manager
  3. Edit Leave module via content management. Create leave types (sick leave, annual leave etc). Create at least on staff group. The group will need members, a manager and an email alertee (usually the manager). Note that participating staff will need to have the firstname, lastname and email address fields completed in their user accounts. This information is input by the intranet administrator
  4. Give the participating staff and managers "read only" permissions on the leave module
  5. The leave module will now appear in the main menu navigation menu.
    • Staff that are in a leave group will be able to apply for leave, and view a chronological list or a calendar view of their leave applications.
    • Staff that are "managers" in a leave group will be able to apply for leave, and view a list or calendar view of their applications. They will also have a be able to manage the leave applications for all staff in their group. Leave applications can be sorted by staff member, by staff group, by leave type or approval status. Managers can edit and approve applications. When a leave application approval status changes the applicant is sent a notification email.
