Brushtail Content Management Guide

Content editing history (undo)

Content history

All content changes are stored by Brushtail. Allowing text changes, images and documents to be available again at the click of a button. All changes are also logged by username, ip address and time.

How long are changes stored?

The intranet administrator can configure the how far back the intranet can "undo" a change. This is set by the administrator by going to Intranet administration > Database cleanup > Content edit history.

This can be set to 1-12 months, or even to DO NOT DELETE.


Paragraph history

Every time a paragraph is has text updated, or images added and documents added this is logged in the Paragraph history.

Every text update, file upload or deletion is logged. The username, time and ip address of the editor is logged. Changes to paragraph text, deleted images and documents can all be restored while stored in the paragraph history.

Page history

If any paragraphs have been deleted then the page acquires a page history.

The username, time and ip address of the editor is logged when a paragraph is deleted or restored. Any deleted paragraphs can be restored using the restore button.

Module history

If any pages have been deleted then the page acquires a module history.

The username, time and ip address of the editor is logged when a paragraph is deleted or restored. Any deleted paragraphs can be restored using the restore button.
