Brushtail Content Management Guide

Editing content pages

Introduction

A user with read privileges may read a content page.
A user with power user privileges may post comments on a content page.
A user with edit privileges may edit a content page.

Main menu

By default a content page accessed through the intranet main menu will display in "normal mode". Clicking on the edit icon will change the page into edit mode. Only existing pages can be edited this way.

Content Management

A user with edit privileges may also edit a content page via the Content management menu. Pages accessed via the content management menu are always in edit mode. Editing via the content management menu also gives you the ability to add/remove pages and change page properties.



To edit a content page click on the edit icon .

Add a widget


Widgets include
  • Calendar widget will display today's events from a calendar.
  • Room bookings widget will display todays bookings.
  • Noticeboard widget will display summary of recent posts.
  • Content widget will display a javascript slideshow of a content page. Only one slideshow on a page is allowed. The content page being shown as slideshow must be enabled as an rss feed. This is done via page properties.

Add a box

Adding a bordered box. The box cannot be moved, however content paragraphs and widgets may be moved in or out of boxes.

Add a column

Adding a bordered column. The column cannot be moved, however content paragraphs and widgets may be moved in or out of cloumns.

Wysiwyg editors

Brushtail has a choice of three different WYSIWYG editors.
  • openWYSIWYG
  • CKEditor
  • NicEdit
The choice of WYSIWYG is configured in includes/config.php

//WYSIWYG editor Values:openWYSIWYG , CKEditor, NicEdit
$WYSIWYG = "NicEdit";

The wysiwyg editors allows content editors to format content without needing to know any html or tags. It is disabled for each module by default and can only be enabled by the intranet administrator. The wysiwyg editors allows html input. This can be a potential security risk. So the wysiwyg editor should only be available to trusted users. The Administrator can enable the wysiwyg by going to Intranet administration > Navigation menu and then edit the relevant module.

RSS feed brief

RSS feeds can be embedded into page content..

[rss_brief]feed url[/rss_brief] will embedd rss content using javascript after content page loads (good for remote content).

[rss_brief_inline]feed url[/rss_brief_inline] will embedd rss via php before content page loads (good for local content).

Will display an rss feed as a bullet list without item descriptions. The list may be kept to specified length using the max parameter.

[rss_brief max=10]feed url[/rss_brief]

RSS feed full

Will display an rss feed with item descriptions.

[rss_full]feed url[/rss_full]will embedd rss content using javascript after content page loads (good for remote content).

[rss_full_inline]feed url[/rss_full_inline] will embedd rss via php before content page loads (good for local content). The list may be kept to specified length using the max parameter.

[rss_full max=10]feed url[/rss_full]

Tabbed RSS

[rss_brief]label1:url1,label2::url2,label3::url3[/rss_brief]

Will display an rss feeds using tabs for a compact display.
Each feed entry is a Tab label name and feed address separated by ::
Each feed entry is the separated by a comma , The max parameter may also be used in this context.

[rss_brief max=10]label1:url1,label2::url2,label3::url3[/rss_brief]

Adding images

Images can be added to a paragraph. Images will be displayed between the title and text of a paragraph. Click on the add picture icon

 

A dialogue box will be displayed you to browse to file location to be uploaded.

 


Adding documents

Documents can be added to a paragraph. Documents will be displayed at the footer of a paragraph. Click on the paperclip icon

A dialogue box will be displayed you to browse to file location to be uploaded. In addition you may put information in the are labelled " "Document metadata keywords". If this field is used then documents may appear in the search results of an intranet keyword search.

Ordering of paragraphs

If a Paragraph ordering of a page has been set to "Custom", then up and down arrows will displayed to move paragraphs up and down the page.

Editor details

The time, ip address and intranet login of the last edit to a paragraph is displayed to the administrator beneath the paragraph title..
